# Device Overrides

Device Overrides let you refine how HADocs interprets specific devices when installation-specific context differs from the default analysis.

Use an override to suppress expected behavior without hiding unrelated findings. Overrides are stored locally and should not be committed when they contain private device information.

Review overrides after device replacement or major integration changes to ensure they still match the installation.

See [Explorer](Explorer.md) for installation context, or return to the [documentation home](../../README.md).
