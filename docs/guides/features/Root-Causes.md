# Root Cause Analysis

Root Cause Analysis groups related symptoms into a smaller set of evidence-backed causes. This prevents one offline device or failed integration from appearing as hundreds of unrelated repair tasks.

Each root cause can connect the likely cause with affected devices, entities, child incidents, estimated repair time, and Potential Health Score gain.

Use Root Cause Analysis to repair the underlying cause before investigating individual symptoms.
