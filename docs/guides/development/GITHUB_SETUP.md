# GitHub setup

Fork the repository, create a focused branch, and open a pull request against the main project repository.

Before requesting review:

- Run the complete test suite.
- Update documentation for user-visible changes.
- Confirm that no tokens, private URLs, generated reports, caches, or local Device Overrides are included.
- Keep commits focused and use clear, imperative summaries.

See [CONTRIBUTING.md](../../../CONTRIBUTING.md) for the complete workflow, or return to the [documentation home](../../README.md).
